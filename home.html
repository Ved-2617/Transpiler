How it works

Five stages.
One unified output.

01
◈

Lexer

The source code string is scanned character-by-character. Numbers, identifiers, keywords, and operators are extracted into a flat token stream with their types and lexemes preserved.

Token Stream
02
⌥

Parser

The token stream is consumed by a recursive-descent parser that understands statements, expressions, conditionals, loops, assignments, and function calls across all four input languages.

AST Nodes
03
⬡

IR Builder

Parsed AST nodes are lowered into a flat Intermediate Representation — a language-neutral list of typed IR nodes. This is the heart of the transpiler, making all code-gen targets equal.

IR Nodes
04
⊗

Code Generator ×4

Four independent code generators walk the IR and emit idiomatic code in Python, C, C++, and Java. Each generator handles types, indentation, semicolons, and language-specific idioms.

×4 Outputs
05
✦

Output & Visualization

Results are streamed back as structured JSON. The browser renders syntax-highlighted code, a token list, the IR table, and a side-by-side diff view — all live.

JSON → UI

About the project

Built from scratch.
No shortcuts.

XPILE is an educational systems project demonstrating how real compilers work — from raw source text all the way to executable-ready output in multiple languages. Every stage is hand-built in C with zero external libraries.

The web layer is intentionally thin: a minimal Python HTTP server pipes stdin/stdout to the C binary, and a pure-HTML/CSS/JS frontend visualizes the results. No frameworks. No dependencies.
